# Module 12 — Architecture

## Overview

CSS architecture determines how styles scale. This module covers methodology comparisons (BEM, utility-first, component-scoped), naming conventions, file organization, and the trade-offs each approach makes.

## Prerequisites

- Module 02: The Cascade (specificity, layers)
- Module 07-08: Flexbox & Grid (layout primitives)

## Lessons

| # | Lesson | Topics |
|---|--------|--------|
| 01 | [The Scaling Problem](01-scaling-problem.md) | Why CSS breaks at scale, specificity wars, naming collisions, dead code |
| 02 | [BEM & Methodologies](02-bem.md) | BEM convention, OOCSS, SMACSS, ITCSS — structured naming |
| 03 | [Utility-First CSS](03-utility-first.md) | Tailwind approach, atomic CSS, trade-offs, when it works |
| 04 | [Component-Scoped Styles](04-component-scoped.md) | CSS Modules, CSS-in-JS, scoped styles, Shadow DOM |
