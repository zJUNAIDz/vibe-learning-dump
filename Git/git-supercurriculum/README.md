
# README.md

Welcome to the Git Supercurriculum.

This is a professional Git curriculum designed for experienced software developers who want to achieve true mastery of Git for team-based, production environments.

## Philosophy

This course treats Git as a distributed data system, not just a set of commands. We focus on building deep mental models from first principles. Our goal is to empower you to reason about any Git scenario, no matter how complex, and to be the person on your team who can solve any version control problem with confidence.

## Who Is This For?

- Experienced developers with years of casual Git usage.
- Engineers who want to understand Git's internals.
- Team leads who need to define and manage robust workflows.
- Anyone who wants to move beyond memorizing commands and truly understand the system.

This course is **NOT** for beginners. A solid understanding of basic Git commands (`add`, `commit`, `push`, `pull`, `branch`, `merge`) is a prerequisite.

## How to Use This Repository

1.  Start with the `00-foundations` module to align on philosophy and mental models.
2.  Proceed through the modules sequentially. They are designed to build on each other.
3.  **Do the exercises and labs.** This is not a passive reading course. The only way to learn is by doing, breaking, and fixing.
4.  Use the `10-appendices` as a reference throughout the course.

## Course Structure

- **/00-foundations**: Core concepts, mental models, and course structure.
- **/01-git-internals**: A deep dive into the object database, the heart of Git.
- **/02-advanced-local-workflows**: Mastering your local repository with tools like `stash`, `reset`, and `worktree`.
- **/03-collaboration-and-remotes**: The art of distributed workflows, tracking branches, and remote management.
- **/04-history-rewriting-and-archaeology**: The power and danger of `rebase`, `cherry-pick`, and history exploration with `bisect` and `blame`.
- **/05-recovery-and-debugging**: The Git survival guide. Using `reflog` and other tools to recover from any disaster.
- **/06-branching-strategies-and-release-workflows**: Architecting robust workflows like GitFlow, GitHub Flow, and Trunk-Based Development.
- **/07-scaling-git-and-performance**: Handling monorepos, large files (LFS), and performance tuning.
- **/08-team-dynamics-and-psychology**: The human side of Git, including code reviews, communication, and conflict resolution.
- **/09-tooling-and-productivity**: Supercharging your workflow with aliases, hooks, and advanced tools.
- **/10-appendices**: A technical reference for plumbing commands, a glossary, and more.
- **/11-capstone-projects**: Large-scale simulations to test your skills in a realistic environment.
- **/12-interview-preparation**: Common and advanced Git interview questions and scenarios.

Let's begin.
