
# Prerequisite Self-Assessment

- **Purpose**: To allow the learner to self-assess their current Git knowledge and confirm they are ready for this advanced curriculum.
- **Estimated Difficulty**: 1/5
- **Estimated Time**: 15 minutes (Quiz)
- **Prerequisites**: Basic Git experience.

---

### Are You Ready for This Course?

This curriculum assumes you are already comfortable with the basics of Git. It is designed for experienced developers, not beginners. Answer the following questions honestly to see if this course is the right fit for you right now.

If you answer "No" to several of these, you may benefit from a more introductory course before tackling this one.

### The Checklist

1.  **Core Commands**:
    - [ ] Can you, without looking it up, explain the primary function of `git init`, `git clone`, `git add`, `git commit`, `git status`, `git push`, and `git pull`?
    - [ ] Have you ever configured your `user.name` and `user.email`?
    - [ ] Do you know how to view the commit history using `git log`?

2.  **Branching and Merging**:
    - [ ] Have you created, switched to, and deleted a branch (`git branch`, `git checkout`/`git switch`)?
    - [ ] Have you ever merged one branch into another (`git merge`)?
    - [ ] Do you understand what a merge conflict is, even if you find them intimidating?

3.  **Remote Repositories**:
    - [ ] Have you cloned a repository from a remote URL (like from GitHub or GitLab)?
    - [ ] Do you understand the concept of a `remote` (e.g., `origin`)?
    - [ ] Have you used `git push` to send your local commits to a remote repository?
    - [ ] Have you used `git pull` or `git fetch` to get changes from a remote repository?

4.  **Workflow Experience**:
    - [ ] Have you used Git as part of a software development project (personal or professional)?
    - [ ] Are you comfortable using the command line?
    - [ ] Do you understand that `git add .` stages all changes in the current directory?

### Mini-Quiz: The "Why"

1.  When you `git add <file>`, what are you *really* doing?
    a) Saving the file.
    b) Marking the file for the next commit.
    c) Adding the file's current content to a "staging area" to be included in the next snapshot.
    d) Uploading the file to the remote.

2.  What is the primary purpose of a branch?
    a) To make a backup of your code.
    b) To isolate a line of development (e.g., a new feature or a bug fix).
    c) To give different developers their own area to work.
    d) To speed up Git.

3.  What does `git commit` do?
    a) It saves your changes.
    b) It creates a new snapshot of the project based on the current state of the staging area (the index).
    c) It sends your changes to your teammates.
    d) It checks for errors in your code.

---
*Answers: 1(c), 2(b), 3(b)*

### Results

- **If you answered "Yes" to all checklist items and got the quiz questions right**: You are perfectly positioned to start this course. You have the foundational experience needed to tackle the advanced topics ahead.

- **If you struggled with a few items**: You can still proceed, but be prepared to spend extra time in the early modules. Pay close attention to the "Foundations" and "Advanced Local Workflows" sections.

- **If most of this was new to you**: This course is likely too advanced for your current level. We highly recommend starting with a beginner-focused Git tutorial (like the official Pro Git book, which is available for free online) and then returning here once you're comfortable with the daily commands. This curriculum will be waiting for you when you're ready to make the leap to expert.
